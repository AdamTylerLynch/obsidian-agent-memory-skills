# Universal Patterns

Language-agnostic and framework-agnostic patterns.

If a pattern is specific to a technology, it belongs in [[domains/Domains|domains]].
If a pattern is specific to one project, it belongs in that project's `patterns/` folder.

## Categories

### Design
- SOLID principles
- Tolerant Reader pattern
- Dependency injection

### Testing
- Unit vs integration test separation
- Fixture reuse strategies

### Security
- Input validation at boundaries
- Credential resolution patterns
